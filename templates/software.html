WordPress Configuration Recommendations (by VPS Tier)

Conservative recommended values to ensure system stability without OOM. You can increase these in Software Management based on your actual needs.

VPS Config Use Case PHP memory_limit upload / post max_input_vars MariaDB pool Redis maxmemory
1C 1G 1 simple blog 128M 32M / 32M 1000 128M 64mb
1C 1G 1 site with many plugins 256M 64M / 64M 2000 128M 64mb
1C 2G 1 e-commerce / Elementor site 256M 64M / 64M 3000 256M 128mb
1C 2G 2-3 simple blogs 256M 64M / 64M 2000 256M 128mb
2C 2G 1 e-commerce + 2 showcase sites 256M 64M / 64M 3000 256M 128mb
2C 4G 3-5 sites (including e-commerce) 512M 128M / 128M 3000 512M 256mb
2C 4G 5+ simple blogs 256M 64M / 64M 2000 512M 256mb
2C 8G Multiple e-commerce / High-traffic sites 512M 128M / 128M 3000 1G 512mb
4C 8G 10+ sites / High concurrency 512M 128M / 128M 5000 1G 512mb
4C 16G Large multi-site / High load 512M 256M / 256M 5000 2G 1gb

General Guidelines

  • post_max_size must be ≥ upload_max_filesize; Nginx client_max_body_size should match
  • max_execution_time is recommended at 300 globally; temporarily increase to 600 for data imports or batch processing
  • MariaDB innodb_buffer_pool_size should be 10%-25% of physical RAM (conservative); MySQL official recommendation is up to 50%-70%
  • Redis maxmemory is about 64-128mb per site; scale proportionally for multiple sites but do not exceed 50% of free memory
  • The main difference between single and multi-site setups is MariaDB pool and Redis; PHP parameters depend mainly on site complexity
