安全设置说明

Fail2ban 配置项

  • 触发阈值（次）

    在统计窗口内，单个 IP 触发特征规则的次数达到此值即自动封禁。默认 5 次。调低更敏感（容易误封），调高更宽松。具体检测特征规则详见「安全防御」页面。

  • 统计窗口（秒）

    用于累计违规次数的时间范围。默认 60 秒。与触发阈值配合使用：例如默认配置即"60 秒内违规 5 次则封禁"。窗口越大，累计越容易触发。

  • 封禁时长（秒）

    首次违规触发封禁后，IP 被防火墙拦截的时长。默认 600 秒（10 分钟）。到期后 fail2ban 自动解封，但若该 IP 在 24 小时内再次违规，将自动升级为 24 小时封禁。

白名单机制

  • 官方白名单

    面板自动从以下官方来源拉取最新 IP 段，加入 fail2ban 白名单，防止误封合法爬虫和 CDN 节点：

    • Cloudflare — 全球 CDN 和反向代理 IP 段（ipv4 + ipv6）
    • Googlebot — Google 搜索爬虫官方 IP 段
    • Bingbot — Bing 搜索爬虫官方 IP 段

    勾选"每周自动更新"后，系统将在每周一凌晨 4:00 自动重新拉取最新 IP。

  • 自定义白名单

    手动添加不会被封禁的 IP 或 IP 段，每行一个。支持 IPv4/IPv6 地址和 CIDR 格式（如 10.0.0.0/8）。自定义白名单不会被官方拉取覆盖。

  • WordPress 安全日志路径白名单

    用于排除搜索引擎、广告平台、站点所有权验证等正常文件。该配置只影响 WordPress 异常访问统计，不影响通用 PHP 网站，不会直接放行或封禁请求。

请求频率限制详解

  • 工作原理：令牌桶算法

    系统为每个 IP 维护一个容量等于「突发缓冲」的令牌桶，每秒自动补充 1 个令牌（由请求频率决定）。请求到达时消耗令牌，耗尽后返回 429。人类浏览有自然停顿（阅读页面），停顿期间桶自动回满；机器人持续不断请求，令牌耗尽后被限制在每秒仅 1 次请求。

  • 请求频率（次/分钟）

    控制长期持续速率，即令牌补充速度。默认 60 次/分钟（= 1 次/秒）。机器人扫描通常远超此值，因此会在数秒内耗尽令牌桶，几乎所有请求都被返回 429。

  • 突发缓冲（次）

    控制令牌桶容量，即页面打开瞬间允许的最大并发请求数。默认 300 次，可覆盖 Elementor + 多插件站点（单页 100+ 请求）同时打开 2-3 个标签页的场景。如需同时打开更多标签页，请设置更大值（如 500-600）。非 Elementor / 轻量站点可适当调低至 30-120。

    增大此值不会延迟拦截机器人——机器人被持续速率（1 次/秒）压制，与突发缓冲大小无关，安全防护不削弱。

  • ⚠ 警告：多次返回 429 的 IP 将被 Fail2ban 自动封禁

    429 响应会被 Fail2ban 规则捕获并计入违规次数（与 wp-login 暴力破解等同级处理）。累计触发阈值后将导致该 IP 被防火墙封锁。请确保突发缓冲足够日常使用，避免正常访客误触发 429 而被拉黑。

  • ✓ 提高突发缓冲不影响拦截机器人

    机器人的特征是持续不断发请求（无人类浏览停顿），会被「请求频率」死死限制在 1 次/秒，令牌桶耗尽后几乎所有请求都返回 429。提高突发缓冲只会给人类访客更宽松的首屏加载空间，安全防护能力完全不受影响。真正的限制来自持续速率，而非突发上限。

  • 如何测试配置是否合适

    用浏览器无痕窗口打开网站，随机浏览 5-10 个页面（含首页、分类页、文章页）。打开开发者工具（F12）→ Network 标签，观察是否有红色 429 状态码。全部 200/304 即表示配置合理。

爬虫限速说明

  • 限速范围

    爬虫限速默认关闭。开启后，每个站点拥有独立的爬虫限速桶；同一站点内命中的常见 Bot UA 共享该桶，普通浏览器不会进入此限速层。

  • 搜索引擎豁免

    Googlebot/Bingbot 只有在来源 IP 命中官方 IP 段时才豁免爬虫限速。伪装成搜索引擎的请求会进入站点的爬虫限速桶。

  • 默认识别的 Bot

    当前默认匹配 Googlebot、Bingbot、Facebook/Meta（facebookexternalhit、facebook、meta-externalagent）、Twitterbot、LinkedInBot、Slackbot、Discordbot、TelegramBot、SemrushBot、AhrefsBot、MJ12Bot、DotBot，以及 UA 中包含 bot、crawler、spider、scraper、scan 的请求。

  • 建议默认值

    默认 30 次/分钟、突发 20 次，主要用于压制多 IP 假爬虫持续扫描，降低 WordPress 动态 404 压力。调整后可观察访问日志中 429 数量和正常抓取情况。
