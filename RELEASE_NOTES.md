# 更新说明

## v1.2.10

### 新增
- 文件管理器新增 HTTPS 远程导入文件功能，可直接将远程备份包导入到网站目录或备份目录，并显示下载进度。
- 远程导入支持通过勾选项允许自签名或不受信任的 HTTPS 证书，便于在受信任服务器之间迁移备份文件。

### 优化
- 文件管理器大文件上传的分片暂存目录改为面板数据目录，避免服务器 `/tmp` 容量较小时上传失败或影响建站。
- 文件上传和远程导入增加更明确的错误提示，空间不足、目标未选择等问题会直接显示具体原因。
- 上传会话增加过期清理，自动清理历史中断上传留下的 `wppanel-upload-*` 暂存目录。
- 远程导入到网站目录后自动设置为站点用户权限，减少导入后还需要手动修复权限的情况。
- 恢复备份前自动清空目标数据库所有表，避免新旧数据表混杂导致冲突。
- 数据库表前缀检测现在会列出所有候选前缀，并优先推荐 wp-config.php 中配置的前缀，多个前缀时会明确提示用户选择。

### 修复
- 修复文件管理器未选择网站或备份目录时仍可上传文件的问题。
- 修复大文件上传中断后残留分片可能占满 `/tmp`，导致后续上传或创建网站失败的问题。

### 安全
- 远程导入仅允许 HTTPS URL，不开放 shell、wget 或任意命令执行。
- 远程导入会阻止访问本机、内网、链路本地和云 metadata 地址，并限制单文件大小和目标磁盘剩余空间。
