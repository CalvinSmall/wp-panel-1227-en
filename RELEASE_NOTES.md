# 更新说明

## v1.2.26

### 网站日志查看与下载

- 网站详情页的日志区域明确标注为“当前日志最近行数预览”，默认显示最近 200 行，避免把预览误认为完整访问日志。
- 新增“下载当前完整日志”入口，可直接下载当天当前日志文件用于本地分析。
- 新增“历史日志下载”列表，展示当前日志和历史轮转日志的文件名、大小、修改时间、状态，并支持逐个下载。
- 访问日志、错误日志和 WordPress 安全日志均支持对应文件列表和下载。

### 日志轮转命名

- 未来生成的站点 logrotate 配置启用日期后缀，历史日志会按日期命名，便于定位具体日期的访问记录。
- 兼容旧版数字轮转文件，例如 `access.log.1`、`access.log.2.gz`。
- 支持新版日期轮转文件，例如 `access.log-2026-06-29`、`access.log-2026-06-29.gz`。
- 已存在的旧日志文件不会被自动重命名，升级后从后续轮转开始逐步使用日期命名。

### 文件下载安全

- 日志下载接口只允许访问当前站点日志目录下的合法日志文件名。
- 拒绝路径穿越、子目录路径、错误日志类型、非法后缀和符号链接文件。
- 日志预览、清空和下载统一使用服务端路径校验，避免前端传入任意文件路径。

### 验证

- 补充日志文件名校验、日志文件列表、日志下载接口、符号链接拒绝和路由注册测试。
- `scripts/verify.sh` 已通过，覆盖 Go 测试、Go vet、构建、diff 检查和配套插件 PHP 语法检查。
